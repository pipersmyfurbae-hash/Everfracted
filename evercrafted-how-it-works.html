Phase 01 — 02

The input
phase. Feeling meets inventory.

Every Evercrafted design begins with two inputs: an emotional prompt — a memory, a mood, an occasion — and your real inventory. The first two engines translate these raw human inputs into structured design signals the rest of the pipeline can act on.

01
Input Layer
Memory Interpreter

Parses your emotional prompt — a sentence, a memory, an occasion — into a structured design brief. Extracts emotion tags, energy levels, palette bias, and symbolic material references that anchor the entire design.

// Input "prompt": "nostalgic and cozy for Christmas" // Output { "emotion": "nostalgia", "energy": 0.4, "palette_bias": "warm", "symbolic_tags": [ "evergreen", "berries", "pinecone" ] }
Emotion Tags Energy Score Palette Bias Symbolic Tags
02
Translation
Emotion Mapper

Converts the emotional brief into actionable design parameters. Maps feelings to colour palettes, contrast ratios, density preferences, and silhouette tendencies — the language the composition engine speaks.

// Emotion → Design signals { "palette": [ "deep_red", "pine_green", "gold" ], "contrast": 0.7, "density_bias": "moderate", "silhouette": "crescent" }
Colour Palette Contrast Ratio Density Bias Silhouette Hint
04
Constraint Layer
Inventory Analyzer

Converts your uploaded stem inventory into structured role pools. Every stem is classified — focal, secondary, accent, filler, or greenery — and the engine calculates average bloom diameter, average stem length, and a flexibility score that influences formula selection.

Role Pools Avg Bloom Diameter Flexibility Score Stem Count

Phase 03 — 04

The genome
phase. Every stem, alive.

In Evercrafted, a stem is not just a flower — it is a design organism. The Floral Genome Engine assigns each SKU a set of traits that determine how it behaves in a composition. The Formula Selector then uses these traits, alongside your inventory and emotional signals, to choose the best structural formula.

03
Core Intelligence
Floral Genome Engine

Every stem in your inventory is represented as a genome — a structured object with role capabilities, bloom dimensions, gesture type, texture, emotion tags, and substitution neighbours. This is what makes Evercrafted a design intelligence, not a random picker.

{ "sku": "96532NT", "name": "Rose Tamora", "role_capabilities": [ "focal", "secondary" ], "bloom_diameter": 3.8, "gesture_type": "cluster", "texture": "velvet", "emotion_tags": [ "romantic", "luxury" ], "substitution_neighbors": [ "english_rose", "garden_rose" ] }
05
Decision Engine
Formula Selector

Scores every formula in the registry across six weighted dimensions. The highest-scoring formula becomes the structural backbone of the design. A confidence score is returned alongside the selection.

// Scoring model score = 0.30 × inventory_feasibility + 0.20 × emotional_match + 0.15 × silhouette_fit + 0.15 × negative_space + 0.10 × complexity + 0.10 × genome_compatibility // Output { "selected_formula": "crescent", "confidence": 0.84 }
Selected Formula Confidence Score Runner-up Formulas
Phase 05 — 07

The placement
phase. Geometry made beautiful.

Once a formula is selected, the engine generates cluster anchors in polar coordinates, divides the wreath into density zones, allocates stems to roles, and solves for the exact position of every single stem — angle, radius, depth, and direction vector.

06
Geometry
Anchor Generator

Transforms formula anchor definitions into precise polar coordinates using seeded randomness. The seed is derived from a hash of your memory input, inventory signature, and formula ID — guaranteeing that identical inputs always produce identical designs.

// Seed generation seed = hash( memory_input + inventory_signature + formula_id ) // → 92384723 // Anchor output { "anchor_id": "primary", "theta": 218.3, "radius": 0.73 }
Polar Coordinates Cluster Anchors Seeded Randomness
07
Spatial Logic
Placement Solver

Assigns every stem a precise angle, radius, depth layer, and direction vector. Collision detection enforces role-specific minimum spacing — focal stems require 1.3× their bloom diameter of clearance, secondary 1.0×, filler 0.6×. The Direction Vector Engine ensures every stem faces the natural arc tangent for organic flow.

// Per-stem placement { "stem_id": "023", "role": "secondary", "angle": 231.5, "radius": 0.72, "depth": 1, "vector": 245 } // Collision rules focal = 1.3 × bloom_diameter secondary = 1.0 × diameter filler = 0.6 × diameter
Angle Radius Depth Layer Direction Vector
08
Depth System
Depth Layering

Every stem is placed on a specific depth layer — from the structural base to the focal crown. This hierarchy ensures the design reads correctly from a distance and builds naturally from the wreath base outward.

Phase 09

The repair
engine. Until it's right.

No design leaves the pipeline with a score below 0.78. The Repair Engine runs up to eight iterative improvement passes — each targeting a specific quality dimension. If the threshold is still not reached, anchors regenerate and the entire placement cycle restarts.

Eight passes.
One standard.

Each repair pass targets a specific quality failure. Passes run in sequence — density first, then cohesion, then silence, then clarity. The engine stops as soon as the composite score clears 0.78.

P1 Density Normalisation
P2 Cluster Cohesion
P3 Silence Arc Enforcement
P4 Focal Clarity
P5–8 Anchor Regeneration (if needed)

Seven metrics.
One composite score.

Phase 10 — Output

Six deliverables.
One pipeline run.

Blueprint JSON

The complete machine-readable design file — every stem with its SKU, role, angle, radius, depth, and vector. Fully versioned and reproducible via seed.

SVG Canvas Preview

A polar-coordinate SVG render of the blueprint — every stem placed at its exact angle and radius. Zoomable, exportable, and embeddable in client presentations.

Builder Guide

A step-by-step construction guide — greenery skeleton first, then focal blooms, secondary, accents, and filler. Written for real hands building a real wreath.

Render Prompt

A production-ready Midjourney prompt — botanical species, colour palette, lighting, texture, and composition style — calibrated to the design's genome and emotion tags.

Inventory Cut List

An exact cut list — every SKU, quantity, role, and cost — ready to hand to a supplier or pull from your inventory. Linked to your Inventory Weaver app.

Score Breakdown

A full seven-metric score report — balance, rhythm, contrast, density, focal clarity, negative space, and inventory efficiency — with the composite score and repair history.

Genome Intelligence

When inventory
runs short.

If a required stem is not available, the Genome Substitution Engine finds the closest match from your inventory using a five-dimension similarity score. Replacements maintain the design's emotional intent, visual weight, and gesture type.

The substitution chain cascades through neighbours until a viable match is found — or the engine flags the gap for manual review.

API Reference

Built for
integration.

POST
/generate-blueprint

Runs the full 10-engine pipeline from a memory prompt and inventory payload. Returns a complete blueprint JSON with score breakdown and render prompt.

POST
/score-blueprint

Evaluates an existing blueprint against the seven-metric scoring model. Returns per-metric scores and a composite score without modifying the design.

POST
/repair-blueprint

Runs the repair engine on a blueprint that has fallen below the 0.78 threshold. Returns the repaired blueprint and updated score breakdown.

POST
/render-blueprint

Generates an SVG canvas preview and a Midjourney render prompt from a blueprint JSON. Accepts custom palette overrides.

POST
/export-blueprint

Exports a blueprint as a PDF builder guide, CSV cut list, or shareable blueprint DNA string. Supports client portal delivery.

GET
/formula-registry

Returns the full list of available composition formulas with anchor definitions, zone configurations, and density curves.

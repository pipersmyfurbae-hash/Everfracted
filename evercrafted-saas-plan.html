Platform Overview

The Vision
Core Model
AI Design Engine

Wraps the composition pipeline in a multi-tenant product with authentication, billing, saved blueprints, and team collaboration.

Primary Value
Reproducible Output

Deterministic seeded randomness ensures identical inputs always generate identical wreaths — versioned, shareable, and resellable.

Market Position
Floral CAD System

Evercrafted is a design intelligence platform — the first procedural composition engine built specifically for professional wreath artisans.

What Evercrafted Produces

Every generation run outputs five deliverables from a single emotional prompt and inventory snapshot: a mathematically balanced wreath layout using polar coordinate placement, a photorealistic Midjourney render prompt compiled from the Floral Genome, a builder-ready step-by-step guide with stem insertion angles, a cut list with SKU quantities per role, and a Blueprint DNA fingerprint for product identification and resale.

The system is not a chat wrapper or a random generator. It is a deterministic engine: seed = hash(memory_input + inventory_signature + formula_id). The same inputs always produce the same design.

Polar Placement System

Every stem is placed at (r, θ) — radius from center and angle from 12 o'clock. The working radius for a 24" wreath sits at r_work = 9.5", with focal blooms at r_work and gesture greenery tucked at r_base = 7.5".

The 60/30/10 rule governs role allocation: 60% base greenery, 30% secondary filler, 10% focal and accent elements. Focal blooms must not exceed 1.5× the base width in diameter.

Blueprint & Design System

Polar Precision
Polar Coordinate Blueprint

Every element is placed at (r, θ) — radial distance from wreath center and angle measured clockwise from 12 o'clock (0°). Canvas coordinates are derived as:

// Polar → Canvas XY conversion
x_canvas = cx + r × sin(θ)
y_canvas = cy − r × cos(θ)

// Working radii for a 24" wreath
r_inner = 7.5"   // inner hole edge
r_outer = 12"    // outermost edge
r_work  = 9.75"  // midpoint — default placement
r_accent = 11.5" // near outer edge
r_base  = 8"     // tucked stems

Focal Point Placement Rules

A wreath typically has 1 primary focal point at 180° (bottom center). Supporting focal points at 120° intervals create a triangular visual anchor — a fundamental principle from the Rule of Odds: group elements in 3s or 5s for natural visual rhythm.

Color Distribution

Minimum angular separation between same-color elements: 45° for small wreaths, 30° for large wreaths. Color flow transitions hues gradually around the ring — never scattered randomly.

Core Design Principles Embedded in the Engine

Balance

Radial symmetry repeats elements at equal angular intervals. Asymmetric balance offsets heavier visual weight with spread and density. The Rule of Odds groups elements in 3s or 5s — odd numbers feel more natural than even.

Rhythm & Flow

A dominant direction (clockwise or counter-clockwise stem lean) is established first. Texture levels alternate — fine (baby's breath), medium (eucalyptus), coarse (large blooms). Color transitions gradually around the ring.

Proportion (60/30/10)

60% base greenery forms the skeleton. 30% secondary filler builds depth. 10% focal and accent elements create the emotional statement. Focal blooms must not exceed 1.5× the base width in diameter.

User Management

Access & Security
Authentication & Authorization

Authentication Methods

Email/password, magic link, Google OAuth for convenience, and optional SSO for enterprise Atelier accounts.

Role-Based Access

At minimum: Owner, Admin, Editor, and Viewer roles at the organization level. Tier-gated feature access enforced server-side — never only in the UI.

Account Features

Profile settings, API key management for external blueprint access, team invitations, seat management, and usage dashboards per tier.

Security & Privacy

Data Protection

TLS in transit, encryption at rest for databases and object storage. Organization scoping on every query and service boundary — tenant data never crosses.

Compliance Controls

Configurable retention policies for blueprint history and uploaded inventory files. PII masking in logs, traces, and error reporting. Rate limits, CAPTCHA on signup, and prompt injection checks.

Enterprise Path

Plan for SOC 2 controls early. Isolate high-risk workflows (bulk export, API access) behind stricter permissioning for Atelier accounts.

Operational Flows

Repair Engine Convergence

Repair runs up to 8 iterative passes. If the minimum target score of 0.78 is not reached, anchors regenerate and density is recalculated. The UI shows live score progress during each pass.

Genome Substitution

When a required stem is out of inventory, the substitution engine finds the nearest genome neighbor using a weighted similarity score: 30% bloom size, 25% gesture type, 20% texture, 15% color family, 10% emotional tags.

Team Collaboration

A teammate opens a previous blueprint, comments or edits the result, duplicates the configuration into a reusable template, and shares it across the workspace. Blueprint DNA ensures the design is always reproducible.

Billing & Metering

Usage events fire after each completed blueprint generation. Base subscription plus usage-based overages for high-volume Studio and Atelier accounts.

Deployment & Scaling

Infrastructure
Hosting & Infrastructure

Frontend

Vercel for the Next.js frontend — edge-optimized, zero-config deployments, and preview environments per branch.

Backend

Containerized Python FastAPI composition engine on AWS ECS/Fargate or Railway for MVP. Node.js orchestration layer co-hosted or as a separate service.

Database

Managed PostgreSQL via Neon, RDS, or Supabase. Redis-backed job queues with BullMQ for repair engine passes and render generation. Object storage (S3 or R2) for blueprint exports and SVG artifacts.

Scalability Path

Start as a modular monolith. Split the AI composition engine, billing, and notification workloads into separate services once throughput or reliability needs justify it.

Monitoring & Reliability

Observability

OpenTelemetry traces, structured logs, error tracking via Sentry, infrastructure dashboards, and token-cost dashboards. SLA alerts on latency and failure rate.

Reliability Controls

Health checks, circuit breakers around the composition engine, dead-letter queues for failed repair passes, nightly backups, and restore drills.

API Integrations

Stripe for subscriptions, usage billing, invoices, and customer portal. Resend or Postmark for auth links and notifications. PostHog for product analytics and feature adoption. Optional Midjourney or image generation API for render output.

Monetization Strategy

Price around the business value of the design output, not the token cost. A professionally structured wreath blueprint that saves 2–3 hours of design iteration and eliminates costly stem waste justifies value-based pricing well above cost-plus margins. Include internal cost controls — per-plan blueprint limits, maximum inventory size, and cached result reuse for repeated prompts — to protect margins at scale.

What does the client do once downloaded?
1
Open blueprint in Evercrafted
The blueprint.json loads into their account dashboard automatically. The SVG canvas renders the design — zones, cluster positions, color palette — so they can see exactly what they bought before touching a stem.
In-app
2
Check inventory against materials list
The materials_list.csv loads into Inventory Weaver (if they have it). It flags what they have, what they need, and shows estimated cost. If they're missing items, the app suggests substitutions that fit the formula.
In-app (Inventory Weaver)
3
Generate a Midjourney concept image
They paste the hero_prompt.txt into Midjourney Discord to see a photorealistic render of the finished wreath before building. If they have Visual Companion, they access the full variation matrix — 9 moods, emotion sweep, overlay PNG — to find the exact look they want.
Midjourney
4
Optionally customize in Design Studio
Studio Pro users can open the blueprint in Design Studio, lock zones they want to keep, adjust cluster positions, swap a flower, or tweak the formula. Every edit is saved as a new version — the original blueprint is never overwritten.
In-app (Design Studio)
5
Print the build guide and build
The build_guide.pdf gives them step-by-step instructions with clock positions ("place eucalyptus base layer, 6:00 to 10:00, stems pointing outward"), layer order (greenery first, then secondary, then focal), and notes for each element. It's designed to be followed at the workbench without a screen.
Physical build
6
Apply client customizations (custom kit blueprints)
If they bought a blueprint with a custom kit — like Petal Calligraphy Ring or Galaxy Seed Wreath — the customization_kit.pdf tells them exactly how to personalize it: what paint to use, how to apply it, what to write, which pieces are swappable. This is what they hand to their end customer along with the finished wreath.
Physical customization
7
Share, sell, and get feedback
They photograph the finished wreath and upload it to Visual Companion (if subscribed) as a feedback loop — tagging what matched the blueprint and what they changed. This data feeds the Adaptive Blueprint Composer, which uses it to evolve future designs. The more they build, the smarter their account gets.
In-app (Visual Companion + ABC)
What does the app need to render each blueprint correctly?
📐
Geometry engine
Polar coordinate system (angle + radius_norm)
Base diameter to scale all elements correctly
Zone angle ranges to constrain element placement
Layer (z-index) values for correct overlap rendering
Scale multipliers per element for size variation
🌸
Floral asset library
SVG icon per flower/greenery type (keyed by SKU)
Color variants per asset (blush_pink, sage, cream...)
Bloom diameter metadata for scale calculation
Role classification (focal, secondary, filler, greenery)
🎨
Canvas renderer
1000x1000 SVG canvas (or Canvas API)
Polar-to-Cartesian coordinate conversion
Rotation per element (rotation_deg field)
Base ring SVG (grapevine, foam, wire, straw)
Cluster grouping for visual coherence
📄
Export engine
SVG → PDF conversion for build guide
Build instruction generator from element order + zones
Materials list aggregator from elements array
Blueprint overlay PNG generator (line-art version)

Product line engine

A single wreath blueprint shouldn't be a dead end — it should be the root of an entire product family. Dynamic Variant Branching turns every blueprint into a living lineage: branch by emotion shift, budget tier, seasonal palette, or client constraint. Each branch stores only what changed. The system resolves the full design at generation time. One root blueprint can produce a budget line, a premium metallic line, a client-specific line, and a seasonal flip — all sharing the same structural DNA.

Merge conflicts when root schema changes — versioned deltas with migration helpers Every delta carries a schema_version field. When root rules change, a migration helper auto-converts old deltas to the new format rather than breaking them. Branches that can't migrate cleanly are flagged as "needs review" rather than silently corrupted
Fixed
Performance on deep lineage resolution — memoized resolveBlueprint with smart invalidation resolveBlueprint() caches the merged config at each node. When a parent delta changes, only that node and its descendants are invalidated — not the whole tree. Deep lineages resolve in milliseconds from cache

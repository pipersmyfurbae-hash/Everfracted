Platform Overview

Core Principle: All modules compile into a single canonical blueprint object. This blueprint is the universal data structure that powers the entire platform.

What Evercrafted Does

Evercrafted is the operating system for wreath design. It combines floral design theory, deterministic geometry, AI-assisted creativity, and inventory optimization into one unified platform.

🧠 Memory Weaver

Input: Memory or emotional prompt

Output: Custom wreath design

Transforms personal stories into floral compositions using emotion-to-color mapping and symbolic flower selection.

📦 Inventory Weaver

Input: User's craft inventory

Output: Designs using only available materials

Solves the "what can I make with what I have" problem. Generates designs constrained by real inventory.

🎨 Design Studio

Input: Manual control + editing

Output: Professional-grade blueprints

Full control interface for designers. Lock zones, adjust clusters, refine placements, export production files.

Memory Weaver Prototype

What It Does

Transforms personal memories and emotional prompts into professionally designed wreaths. The system interprets emotion, selects symbolic flowers, chooses appropriate composition formulas, and generates a complete blueprint.

Why It Matters

Marketing hook: "This wreath was designed from my grandmother's memory." Creates shareable, emotional content that drives viral growth.

Business value: Lowers barrier to entry. Non-designers can create meaningful wreaths instantly.

User Flow

  1. User enters memory or emotional prompt (text)
  2. System extracts core emotions (AI analysis)
  3. Emotion mapped to color palette + floral symbols
  4. System selects composition formula based on emotion tone
  5. Wreath blueprint generated using deterministic geometry
  6. User sees SVG preview + build instructions
  7. Export options: SVG, PDF, blueprint JSON
Memory Input Interface

Large text area with prompts like:

"Tell us about a memory you'd like to preserve..."

"What emotion should this wreath capture?"

Engine Logic

Step-by-Step Process
  1. Emotion Extraction (AI): LLM analyzes text and returns emotion array: [joy: 0.8, nostalgia: 0.6, calm: 0.4]
  2. Color Selection: Map emotions to color families using lookup table
  3. Floral Symbolism: Select flowers with matching symbolic meanings
  4. Formula Selection: Choose composition type based on emotion intensity/balance
  5. Generate Blueprint: Run deterministic placement engine
  6. Render + Export: Create SVG visualization and build guide

Inventory Weaver Prototype

What It Does

Generates wreath designs using ONLY materials the user currently has in inventory. Solves the most common crafter question: "What can I make with what I already have?"

Why It Matters

Pain point solved: Crafters waste time guessing what designs are possible with limited supplies.

Business value: This alone could be a standalone SaaS product. High retention potential.

User Flow

  1. User uploads/enters inventory (manual or CSV import)
  2. System categorizes items by role (focal, secondary, filler, greenery)
  3. User requests design ("use what I have")
  4. System checks available quantities and selects appropriate formula
  5. Blueprint generated respecting inventory constraints
  6. System shows inventory depletion (what gets used)
  7. Export SVG, build guide, and updated inventory

Engine Logic

Constraint-Based Design Generation
  1. Inventory Analysis: Count available stems by role (focal: 12, secondary: 20, filler: 30, greenery: 40)
  2. Formula Selection: Choose formula that fits available quantities (if low on focal → use "focal burst" or "crescent")
  3. Placement with Constraints: Engine places elements and checks: required_stems <= available_stems
  4. Automatic Adjustment: If constraint violated, reduce cluster density or switch formula
  5. Usage Report: Return list of items used and updated quantities

Formula Registry Prototype

What It Is

A library of 12 composition formulas. Each formula defines cluster placement rules, zone structures, and aesthetic principles.

The 12 Formulas

1. Crescent

Shape: Asymmetrical arc, heaviest at bottom

Zones: 1 focal (bottom), 1 secondary (arc), greenery trails

Classic

2. Double Echo

Shape: Two mirrored clusters at 4 and 8 o'clock

Zones: 2 focal, bridge greenery

Symmetrical

3. Bottom Heavy

Shape: Dense base, sparse top

Zones: Large focal zone 5-7 o'clock, minimal top

Gravity

4. Side Sweep

Shape: Diagonal flow from 2 to 8 o'clock

Zones: Linear zone structure, directional greenery

Directional

5. Open Arc

Shape: Negative space at top, bloom arc at bottom

Zones: Silence zone 10-2, dense bloom zone 4-8

Modern

6. Full Halo

Shape: Even distribution, no negative space

Zones: 8 equal zones, radial symmetry

Traditional

7. Split Garden

Shape: Two distinct halves (colors/textures)

Zones: Left/right split at 12-6 axis

Contrast

8. Focal Burst

Shape: Single large focal point with radiating accents

Zones: 1 large focal, accent rays

Dramatic

9. Laddered Rhythm

Shape: Stepped clusters at 2, 5, 8, 11 o'clock

Zones: 4 focal zones, rhythmic spacing

Rhythm

10. Radial Burst

Shape: Spokes radiating from center

Zones: 6-8 linear zones from center outward

Geometric

11. Bow Anchor

Shape: Large bow/ribbon at top or bottom, flowers opposite

Zones: Bow zone + complementary focal zone

Seasonal

12. Inventory Salvage

Shape: Adaptive formula using whatever is available

Zones: Dynamic zone creation based on inventory

Adaptive

Formula Selection Logic

How Engine Chooses Formula
Context Formula Selected Reason
Emotion: serene, calm Crescent or Open Arc Negative space = calm
Emotion: joyful, celebratory Full Halo or Radial Burst Abundance = joy
Emotion: nostalgic, romantic Double Echo Symmetry = balance/memory
Low inventory (focal < 10) Focal Burst or Crescent Efficient focal use
High inventory (all roles > 20) Full Halo or Split Garden Can support density
Mismatched inventory Inventory Salvage Adaptive approach

Key Sections Explained

Base Object

Defines the physical wreath form. All measurements in inches.

  • type: grapevine, foam, wire, straw
  • outer_diameter: Total wreath diameter
  • inner_diameter: Open center diameter
  • thickness: Depth of base material
Zones Array

Each zone defines a region with placement constraints.

  • angle_start/end: Degrees (0° = top, clockwise)
  • radius_min/max: Normalized (0 = inner edge, 1 = outer edge)
  • allowed_roles: What element types can go here
  • density_cap: Max fill percentage (0-1)
  • locked: If true, zone cannot be edited in Studio
Elements Array

Each element is a single flower/greenery stem with exact placement.

  • angle_deg: Polar angle (0-360)
  • radius_norm: Normalized radial distance (0-1)
  • rotation_deg: Stem rotation for natural look
  • scale: Size multiplier (0.5 = half size, 1.5 = 150%)
  • layer: Z-index for overlap (1 = back, higher = front)
  • cluster_id: Groups elements that belong together
Scores Object

Algorithmic evaluation of design quality (0-1 scale).

  • balance: Visual weight distribution
  • rhythm: Repetition and flow
  • proportion: Size relationships
  • contrast: Color/texture variety
  • silhouette: Outer shape clarity
  • inventory_fit: How well design uses available materials

Wreath DNA Engine Concept

Future Module: This is the "10x upgrade" — a genetic encoding system that would make Evercrafted impossible to replicate. To be discussed separately.

Concept Overview

Instead of just storing blueprints as static snapshots, the DNA Engine would encode the generative rules that created each design. This allows:

  • Parametric variation: "Make this wreath bigger/smaller/fuller"
  • Style transfer: "Use this wreath's structure but with fall colors"
  • Design breeding: Combine traits from two wreaths
  • Marketplace intelligence: Understand what makes designs successful

What DNA Would Encode

Gene Encodes Example Value
symmetry Radial vs asymmetric balance 0.2 (highly asymmetric)
density Overall fullness 0.75 (quite full)
negative_space Intentional empty areas 0.3 (moderate silence)
focal_dominance How strong focal point is 0.9 (very dominant)
color_harmony Monochrome vs rainbow 0.6 (analogous palette)
texture_variance Smooth vs wild textures 0.8 (high texture mix)

System Architecture Ready

Technology Stack

Frontend

  • Framework: Next.js 14 (React)
  • Language: TypeScript
  • State: Zustand
  • Graphics: SVG + Canvas API
  • Styling: Tailwind CSS

Backend

  • Framework: Python FastAPI
  • Geometry Engine: NumPy + custom algorithms
  • Blueprint Engine: Pure Python
  • Scoring Engine: Python scoring functions

Database

  • Primary: PostgreSQL
  • Tables: users, inventory, projects, blueprints
  • JSON columns: For blueprint storage

Storage & AI

  • Object Storage: S3-compatible
  • LLM: OpenAI GPT-4 (emotion analysis only)
  • Images: User uploads + floral library

Decision Why

Polar coordinates for placement Natural for radial wreath structure

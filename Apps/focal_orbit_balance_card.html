Every blueprint generated gets smarter balance automatically — no user action needed

Every blueprint the platform generates already uses polar coordinates for placement. Focal Orbit Balance adds a gravitational scoring layer on top — treating the wreath as a field of visual mass orbiting one or more focal points. Heavier elements (large blooms, high-contrast colors, rich textures) pull the eye more strongly. The engine now knows this and balances the field automatically. The result: every blueprint feels more intentionally designed, not just algorithmically placed.

How visual mass is computed
Focal orbit — primary bloom or bow, allowed to exceed adjacent orbits by up to 40%
Mid orbit — secondary florals, mass must stay within 20% of focal
Outer orbit — fillers and greenery, distributed in 3 energy zones (rule of odds)
Secondary nodes — optional at 120°–160° offsets, creating dynamic orbit triangles
mass = base × ((size + contrast + texture) / 3)

Fixed
Over-correction ruins intentional asymmetry — blueprint type flags hard-exempt specific zones Crescent blueprints exempt the lower arc from balancing. Heavy base blueprints exempt the bottom third. Side sweeps exempt the diagonal band. The engine knows what's intentional and never "fixes" it
Fixed
Computation cost — hard cap at 12 segments, greedy incremental placement Scoring runs incrementally per element placement, not from scratch each time. Hard cap of 12 segments per orbit keeps the scoring function lightweight regardless of blueprint complexity

Learned parameters need data — ships with hand-tuned defaults mapped to emotion profiles Formal orbit maps to calm and serene. Wild orbit maps to joyful and dramatic. Minimal orbit maps to modern and everyday. Learning layer activates in v2 once real usage data exists — no cold start problem
Visual mass model Every element gets a computed mass score — size, color contrast, and texture combined — so the engine knows what the eye sees as heavy or light
Concentric orbit scoring 4 orbits (inner, mid, outer, edge) each with target mass bands — adjacent orbits can't differ by more than 20%, preventing heavy-ring empty-ring artifacts
Rule-of-odds energy zones High-mass segments group into visual triplets around the ring — one main energy zone, two supporting — creating natural rhythm without forced symmetry
Studio Pro heatmap overlay Designers see a live mass heatmap over the canvas — bright where mass is heavy, cool where it's light — with sliders for orbit strictness and focal dominance
Secondary focal nodes Optionally place 1–2 secondary focal points at 120–160° offsets from the primary — creating dynamic triangular orbit layouts for complex multi-focal designs
Blueprint type exemptions Crescent, heavy base, and side sweep blueprint types flag specific orbit zones as intentionally unbalanced — the engine never corrects what is meant to be asymmetric
